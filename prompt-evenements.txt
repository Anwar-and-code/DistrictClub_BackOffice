Ajoute la fonctionnalité complète de gestion des **Événements** dans le backoffice. Les tables Supabase existent déjà. Respecte strictement les patterns existants du projet.

## Contexte DB Supabase (project_id: vslisxnahktqaifdurcu)

Les tables suivantes existent déjà en base :

**Table `events`** :
- id (uuid PK), title (text NOT NULL), subtitle (text), description (text NOT NULL), long_description (text)
- category (text NOT NULL — enum: 'TOURNOI', 'FORMATION', 'SOCIAL', 'ANIMATION', 'COMPETITION', 'AUTRE')
- status (text NOT NULL — enum: 'DRAFT', 'PUBLISHED', 'CANCELLED', 'COMPLETED')
- start_date (timestamptz NOT NULL), end_date (timestamptz)
- location (text NOT NULL), cover_image_url (text)
- is_featured (boolean default false), display_order (int default 0)
- tags (text[]), price_info (text), is_free (boolean default true)
- contact_phone (text, default '+2250799998888') — numéro de contact affiché dans l'app mobile
- created_by (uuid FK → auth.users), created_at, updated_at

**Table `event_images`** :
- id (uuid PK), event_id (uuid FK → events ON DELETE CASCADE)
- image_url (text NOT NULL), caption (text), display_order (int default 0)

**Storage bucket** : `event-images` (public, 5MB max, jpg/png/webp/gif)

## Ce qu'il faut faire

### 1. Types — `src/types/database.ts`
Ajouter les interfaces `Event`, `EventImage`, `EventCategory`, `EventStatus` en suivant le style des types existants (Terrain, Reservation, etc.)

### 2. Service — `src/lib/services/events.ts`
Créer le service CRUD complet en suivant exactement le pattern de `src/lib/services/terrains.ts` :
- `getEvents()` — select `*, event_images(*)` ordonné par `display_order` puis `start_date`
- `getEventById(id: string)` — avec join `event_images(*)`
- `createEvent(event)` — insert
- `updateEvent(id, updates)` — update
- `deleteEvent(id)` — delete (cascade supprime les images)
- `uploadEventImage(file: File)` — upload dans le bucket `event-images`, retourne l'URL publique
- `deleteEventImage(imageId: string)` — supprime de la table event_images
- `addEventImage(eventId, imageUrl, caption?, displayOrder?)` — insert dans event_images

### 3. Hook — `src/hooks/use-events.ts`
Créer le hook TanStack React Query en suivant exactement le pattern de `src/hooks/use-terrains.ts` :
- `eventKeys` (query key factory)

- `useCreateEvent()`, `useUpdateEvent()`, `useDeleteEvent()`
- `useUploadEventImage()`, `useAddEventImage()`, `useDeleteEventImage()`
Exporter dans `src/hooks/index.ts`
